# PASCO .cap 檔案解析與力學分析技術規格書

本文件記錄了 PASCO 力板專有格式 (.cap) 的逆向工程結果，以及基於開源工具 `JumpMetrics` 實現的自動化跳躍分析邏輯。

---

## 1. 檔案格式解析 (Reverse Engineering)

### 1.1 容器結構
- **格式類型**: 標準 ZIP 容器。
- **元數據**: `main.xml` 包含所有感測器配置、實驗 Run 的編號以及二進位數據檔案的映射。
- **原始數據**: 存放在 `data/` 目錄下的 `.tmp` 檔案，採二進位格式儲存。

### 1.2 二進位數據編解碼 (Binary Decoding)
- **記錄長度**: 每個數據點固定為 **12 bytes**。
- **數據類型**: 
  - Bytes 0-3: 序列號/索引 (Integer, Little-endian)。
  - **Bytes 4-11: 力量值 (Double Precision Float, IEEE 754, Little-endian)**。
- **單位**: 牛頓 (Newton, N)。

### 1.3 設備辨識邏輯 (Device Identification)
- **挑戰**: 每次實驗產生的 `ChannelIDName` 是動態的。
- **解決方案**: 使用 `main.xml` 中的 `DataSource` 節點下的 **`EUID`** 作為唯一硬體標識符。
- **命名映射**: 腳本會自動將穩定出現的 EUID 排序並映射為 `Plate 01` 與 `Plate 02`，確保左右腳數據一致。

## 2. 力學分析算法 (Biomechanical Analysis)

本系統第二階段採用開源專業庫 `JumpMetrics` 進行垂直跳躍分析。

### 2.1 信號處理
- **數位濾波**: 使用四階 Butterworth 低通濾波器，截止頻率設定為 **20Hz**，以去除高頻電訊干擾。
- **體重計算**: 自動截取實驗前 1.0 秒（靜止期）的平均力量作為系統體重 (System Weight)。

### 2.2 相位偵測演算法
- **動作開始 (SoM)**: 採用 **5SD 法**。當總力量偏離靜止期平均值達 5 倍標準差，且持續一定時間後，判定為動作開始。
- **離地瞬間 (Takeoff)**: 當力量下降至 **10~30N** 閾值以下。
- **著地瞬間 (Landing)**: 當力量重新上升並超過設定閾值。
- **其餘事件點**: 
  - **重心最低點**: 位移曲線的最小值。
  - **最大向心/離心功率**: 力量與速度乘積的波峰/波谷。

### 2.3 運動學推導 (Kinematics)
- **加速度 (a)**: $a(t) = (F(t) - Weight) / mass$
- **速度 (v)**: 使用梯形積分法對加速度進行一次積分。
- **位移 (s)**: 對速度進行一次積分。
- **功率 (p)**: $P(t) = F(t) \times v(t)$

## 3. 分析誤差評估 (Validation)

經由 20 組樣本與實驗室原廠黑盒子軟體比對，結果如下：
- **時間相位誤差**: 平均 < 7ms。
- **數值誤差 (力/速度)**: 平均誤差 < 1%。
- **位移誤差**: 約 1~2 cm（主因來自於積分漂移）。

## 4. 維護與改進建議
1. **濾波器調整**: 若數據雜訊過大，可將 `analyze_jump_v2.py` 中的 `FILTER_CUTOFF` 調低。
2. **體重校正**: 對於靜止期不穩定的受試者，建議手動校準體重值。
3. **空中補償**: 未來可加入自由落體公式以完善空中階段的速度曲線。

# IMOTEK Jump Measure (跳躍高度測量工具)

本工具利用 IMOTEK/Xsens IMU 感測器收集的高頻加速度與四元數數據，自動計算垂直跳躍的高度與相關運動參數。

### 🧠 核心演算法
- **座標轉換**: 利用四元數將感測器局部座標（Sensor Frame）轉換為地球慣性座標（Earth Frame），並扣除重力加速度得到自由加速度。
- **DTW 動作識別**: 使用 **Dynamic Time Warping (動態時間規整)** 演算法，將原始加速度信號與標準跳躍模板進行比對，自動精準識別跳躍的起始點（Take-off）與落地點（Landing）。
- **信號濾波**: 整合 Butterworth 低通與高通濾波器，消除感測器雜訊與積分漂移（Drift）。

### 📏 計算方法
工具提供兩種高度計算路徑以供對比：
1. **位移積分法 (Displacement Method)**: 對加速度進行兩次積分得到位移曲線。
2. **飛行時間法 (Flight Time Method)**: 根據 $h = \frac{1}{8} g t^2$ 公式，利用空中的飛行時間推算高度。

## 📂 資料夾結構
- `input/`: **[輸入]** 放置從感測器匯出的原始 CSV 檔案。
- `align_std/`: 存放 DTW 比對用的標準動作模板 (`standard_data.csv`)。
- `output/`: **[輸出]** 自動為每個檔案建立資料夾，包含高度報表 (CSV) 與對比圖 (PNG)。
- `dtw/`: DTW 核心運算模組。

## 快速開始

1. **環境準備**：請確認電腦已安裝 Python 3，並安裝必要套件：
   ```bash
   pip install pandas numpy scipy matplotlib
   ```
2. **準備資料**：
   - 將導出的原始感測器 CSV 檔案放入 `input/` 資料夾。
   - 確認 `align_std/standard_data.csv` 模板檔案存在（用於 DTW 跳躍偵測）。
   - 確認 `dtw/` 模組資料夾在根目錄下。
3. **執行程式**：
   雙擊執行 `jump_measure.py` 或在終端機輸入：
   ```bash
   python jump_measure.py
   ```
4. **檢查結果**：
   處理完成後，程式會在 `output/` 資料夾下替每個檔案建立專屬資料夾，包含高度報表 (CSV) 與兩張分析圖表。

## 資料格式規範 (Initial Data)

輸入的 CSV 檔案必須符合以下 Xsens DOT 導出格式：

*   **取樣頻率**：預設為 **60 Hz**。
*   **首行標記**：第一行需為 `sep=,`，第二行為標題行。
*   **必要欄位**：
    *   `PacketCounter`: 封包計數器（用於確認資料連續性）。
    *   `Acc_X`, `Acc_Y`, `Acc_Z`: 加速度計數據（單位：m/s²）。
    *   `Quat_W`, `Quat_X`, `Quat_Y`, `Quat_Z`: 四元數數據（用於世界座標轉換）。

## 演算法說明

本工具依照以下物理邏輯自動化分析數據：

### 1. 座標系轉換與預處理 (Step 1 & 2)
*   **地球座標轉換**：利用感測器的四元數，將設備座標系下的加速度投影至地球座標系，並減去重力常數（約 9.81 m/s²）得到垂直方向的「自由加速度」。
*   **濾波處理**：對加速度進行低通濾波（25Hz）去除高頻雜訊。對積分後的速度進行高通濾波（0.4Hz）以消除隨時間產生的積分偏移（Drift）。

### 2. 跳躍區間偵測 (Step 3)
*   **DTW 特徵比對**：使用「動態時間校正（Dynamic Time Warping）」演算法，將標準跳躍的速度特徵模板在數據流上滑動比對。
*   **自動過濾**：透過曼哈頓距離與垂直速度閾值，過濾掉非跳躍的雜訊動作。
*   **起落點約束**：程式會找出最大向上速度點作為「起跳點」，並在起跳點之後尋找最大向下速度點作為「落地點」，確保滯空時間為正值且符合生物力學。

### 3. 跳躍高度計算法
程式提供兩種高度指標供研究比對：

1.  **高度指標 A (height_by_displacement)**:
    *   **原理**：對垂直速度進行二次積分得到垂直位移。
    *   **公式**：$h = \max(\text{Displacement}) - \text{Displacement}_{\text{take\_off}}$
    *   **優缺點**：直觀反映感測器計算的運動軌跡，但容易受長時間飄移影響。
2.  **高度指標 B (height_by_flight_time)**:
    *   **原理**：利用起跳與落地的時間差計算空中滯空時間（Flight Time, $t$）。
    *   **公式**：$h = \frac{1}{8} g t^2$ (假設上升與下降時間相等)
    *   **優缺點**：計算簡單且通常較穩定，但若起落點偵測誤差 1 frame (1/60s)，高度會有些微變動。
